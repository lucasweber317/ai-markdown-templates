# Glossary

> This file defines recurring product terminology to ensure consistency across Epics.

---

## Terms

**Instructions:** Add terms here that are used repeatedly in your product context and need consistent definitions. Focus on product-specific terminology, internal codenames, technical concepts, and acronyms that might be ambiguous.

**Examples of terms to include:**

- **Epic**: A structured initiative delivering measurable product value, typically spanning multiple user stories.
- **Primary Persona**: The main user type or role most impacted by a feature or initiative.
- **Success Metric**: A measurable indicator used to determine whether an initiative achieved its intended outcomes.
- **Workspace**: A collaborative container where team members can organize projects, documents, and conversations with shared permissions.
- **Canvas**: The main editing interface where users create and manipulate content in real-time with other collaborators.
- **Organization Admin**: User role with full administrative privileges including billing, user management, and security configuration.
- **SSO**: Single Sign-On - authentication method allowing users to access the platform using their organization's identity provider (e.g., Okta, Azure AD).
- **RBAC**: Role-Based Access Control - permission model where access rights are assigned based on user roles rather than individual users.
- **Sync Conflict**: Occurs when multiple users edit the same content simultaneously; resolved through operational transformation or last-write-wins strategy.
- **Retention Policy**: Time-based rule determining how long data (files, logs, versions) is stored before automatic deletion.

**What NOT to include:**
- ❌ Generic terms everyone knows ("user," "feature," "bug")
- ❌ One-off terms only used in a single Epic
- ❌ Full dictionary definitions—keep it concise and product-specific
