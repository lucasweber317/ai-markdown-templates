# Glossary

Product terminology for consistent language. Add only frequently used terms.

---

## Release Tiers

- **T1**: Market-Level Launch (full cross-functional, executive involvement)
- **T2**: Coordinated External Release (structured planning, no executive)
- **T3**: Targeted Release (lightweight coordination, limited external)
- **T4**: Minimal Communication Update (changelog-level only)

## Product Stages

- **In Development**: Feature building (5-6 weeks pre-launch)
- **Feature Complete**: Refinement phase (3-4 weeks pre-launch)
- **Feature Freeze & Testing**: Testing only, no new features (2 weeks pre-launch)
- **Production Ready**: Staged for deployment (3 days pre-launch)
- **Launch**: Production deployment and communication

## Roles

- **PMM**: Launch coordination, messaging, tier-based planning
- **PM**: Feature truth, scope, readiness criteria
- **Engineering**: Deployment, technical readiness, validation
- **Marketing**: External communication execution
- **Sales**: Enablement consumption, field feedback
- **Customer Success**: Account notifications, adoption support
- **Support**: Documentation, FAQs, support readiness

## Other Terms

- **Release Brief**: Input specifying tier, scope, personas, dependencies
- **Release Checklist**: Generated timeline with responsibilities and communication plan
- **Feature Flag**: Controlled rollout and rollback mechanism
- **Communication Channel**: Customer/market communication medium
- **Enablement**: Training and materials for Sales/CS teams
