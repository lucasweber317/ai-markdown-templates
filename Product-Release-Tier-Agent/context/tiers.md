# Release Tiers – Definitions and Launch Intensity Model

This document defines the release tiers used to determine launch intensity, communication scope, and cross-functional involvement.

The Release Tier Agent must use this document to:

- Scale checklist depth appropriately
- Adjust cross-functional coordination
- Align communication scope with impact
- Prevent over- or under-scoping launch activities

---

## Overview of Tiers

We use four release tiers categorized as T1–T4:

- **T1 – Market-Level Launch**
- **T2 – Coordinated External Release**
- **T3 – Targeted Release**
- **T4 – Minimal Communication Update**

Each tier reflects business impact, customer visibility, and revenue influence — not technical complexity alone.

---

# T1 – Market-Level Launch

## Definition
A high-impact launch that significantly affects company positioning, revenue, or competitive standing.

## Typical Characteristics
- New product or major expansion
- Entry into a new market or segment
- Executive visibility
- High competitive sensitivity
- Broad customer communication required

## Launch Intensity
- Full cross-functional coordination
- External visibility prioritized
- Structured timeline with extended preparation window
- Executive involvement required

## Required Communication Channels
- Release Notes: Yes
- In-product message: Yes
- Customer email: Yes
- Website update: Yes (dedicated landing page or major section)
- Sales enablement: Yes (comprehensive deck, demo, competitive positioning)
- Supporting content: Yes (blog posts, case studies, customer stories, webinars)
- Press / PR: Yes (press release, media briefings, analyst outreach)

---

# T2 – Coordinated External Release

## Definition
A significant feature or capability that meaningfully improves product value or competitiveness but does not redefine company strategy.

## Typical Characteristics
- Major feature addition
- Strong customer demand
- Sales enablement required
- Broad but controlled communication

## Launch Intensity
- Coordinated cross-functional planning
- External communication required
- Standard preparation timeline
- No executive-level launch involvement required

## Required Communication Channels
- Release Notes: Yes
- In-product message: Yes
- Customer email: Yes
- Website update: Yes (feature page update or new section)
- Sales enablement: Yes (enablement deck with use cases and demo flow)
- Supporting content: Optional (blog post or knowledge base article if relevant)
- Press / PR: No

---

# T3 – Targeted Release

## Definition
A meaningful enhancement that improves usability, performance, or fills a feature gap without materially shifting market positioning.

## Typical Characteristics
- Feature enhancement
- UX improvement
- Customer-requested capability
- Limited competitive implications

## Launch Intensity
- Lightweight coordination
- Limited external communication
- Focused enablement
- Short preparation window

## Required Communication Channels
- Release Notes: Yes
- In-product message: Yes (for affected users only)
- Customer email: Optional (only if workflow impact expected)
- Website update: No (or minor documentation update only)
- Sales enablement: Optional (brief update or FAQ)
- Supporting content: No
- Press / PR: No

---

# T4 – Minimal Communication Update

## Definition
Small improvements, bug fixes, backend enhancements, or minor UI adjustments.

## Typical Characteristics
- Low customer visibility
- No strategic positioning change
- Primarily operational impact

## Launch Intensity
- Minimal coordination
- In-product or changelog-level communication only
- No campaign-level activity

## Required Communication Channels
- Release Notes: Yes (public changelog or release notes)
- In-product message: No
- Customer email: No
- Website update: No
- Sales enablement: No
- Supporting content: No
- Press / PR: No

---

# Tier Governance

Tier is declared in the release brief.

The Release Tier Agent must use the specified tier and must not independently determine or override it.

If Tier is missing:
- Halt checklist generation.
- Request clarification.

If the described scope clearly conflicts with the declared Tier:
- Flag for human review.
- Do not reclassify autonomously.

---

# AI Agent Scaling Rules

When generating a release checklist:

1. Use the declared Tier exactly as written.
2. Scale checklist depth proportionally.
3. Do not include higher-tier activities in lower-tier launches.
4. Do not introduce campaign-level work for T4 releases.
5. Prefer right-sized execution over maximal activity.

The objective is proportional launch execution aligned to declared Tier.
